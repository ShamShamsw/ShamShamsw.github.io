Full project description

In a compact two-hour session I created and documented a small, practical prototype inside the ShamShamsw/ChatGTP-Prompt-Engineering repository focused on prompt engineering for generative models. The work concentrated on delivering clear, reproducible notebook examples that demonstrate key prompt patterns, evaluation checks, and usage notes so others can run, review, and extend the experiments immediately.



What I set out to do

  • Define a clear 2-hour goal: produce one or more runnable Jupyter notebooks that demonstrate practical prompt-engineering techniques and include explanatory notes.
  • Prioritize the minimum viable demonstration that communicates the idea and is reproducible by others.


What I actually did (approximate times)

  • Scoping and plan: 10-15 minutes to clarify purpose, audience, and success criteria.
  • Implementation: 60-75 minutes to create and iterate on notebooks with runnable examples, prompts, and explanations.
  • Validation and fixes: 10-15 minutes to run notebooks, verify outputs, and address issues.
  • Documentation and README or site copy: 10-15 minutes to add usage instructions for reproducibility.


Technical summary

  • Primary format: Jupyter Notebook (100% of repository language composition).
  • Languages and tools: Python within notebooks, Jupyter ecosystem, local runtime for testing.
  • Files added or changed: one or more notebooks with prompt examples and explanatory markdown.
  • Tests: manual smoke tests by executing notebook cells and verifying outputs.
  • Tooling: Jupyter Notebook or JupyterLab, local Python environment, git for commits.


Outcome and deliverables

  • Runnable Jupyter notebooks demonstrating prompt engineering patterns and example prompts.
  • Concise in-notebook documentation describing intent, how to run the examples, and recommended follow-ups.
  • A small, focused commit history suitable for review and inclusion in the site.


Challenges and tradeoffs

  • Timeboxed to two hours, so emphasis was on clarity and reproducibility rather than exhaustive coverage.
  • No automated tests or extensive performance or edge-case testing added in this session.


Next steps (recommended)

  • Add clearer filenames and a top-level README with links to each notebook if multiple notebooks exist.
  • Add automated checks that execute notebooks or convert them to tests (for example, nbval).
  • Polish narrative and visuals in the notebooks, and add example outputs or screenshots for the web page.
