D&D Campaign Notes

A polished, easy-to-navigate collection for running and documenting campaigns

Description

This repository centralizes campaign information including session notes, NPCs, locations, characters, maps, and one-off encounters in a lightweight, human-readable format. It is designed for Game Masters who want a structured place to store and quickly retrieve or share information during play.

Back to Home GitHub Repository


Key accomplishments

  • Established a consistent structure for sessions, characters, locations, and assets.
  • Adopted Markdown so notes are readable in any editor and easy to share.
  • Created templates for sessions, NPCs, and character summaries.
  • Organized media and map assets for quick reference and export.
  • Added a concise README and contributing guidance for collaborators.


Suggested time breakdown

  • Planning and structure design: 2-4 hours.
  • Templates and initial content: 6-10 hours.
  • Collecting and organizing assets: 2-6 hours.
  • Documentation: 2-4 hours.
  • Iteration and polish: 1-3 hours.

Total (first pass): approximately 13-27 hours depending on scope and asset collection.



Repository structure

.
├── README.md                # Project overview and usage notes
├── LICENSE                  # Optional license for sharing
├── notes/                   # Primary campaign notes
│   ├── sessions/            # Session-by-session notes (YYYY-MM-DD-title.md)
│   ├── encounters/          # One-off encounters and stat blocks
│   └── lore/                # World lore, factions, timelines
├── characters/              # Player characters and important NPCs
│   ├── pc/                  # Player character sheets or summaries
│   └── npc/                 # Notable NPCs with motivations and hooks
├── locations/               # Towns, dungeons, landmarks
├── maps/                    # Map images and source files
├── assets/                  # Images, icons, tokens
├── templates/               # Markdown templates
├── tools/                   # Utility scripts
└── docs/                    # Supporting documentation


Quick start

  1. Clone the repository.
  2. Copy a template from templates into the appropriate folder.
  3. Name session files as YYYY-MM-DD - Short Title.md for easy sorting.
  4. Store images in maps or assets and reference them from Markdown.


Contributing

  • Use existing templates for consistency.
  • Submit improvements as pull requests or open issues.
  • Keep filenames and folder structure consistent.
