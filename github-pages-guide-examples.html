📂 GitHub Pages Guide — Example Templates & Starter Files

Ready-to-use starter templates and configuration files to help you launch your GitHub Pages site in minutes.

📄 Files Included

This examples folder ships two ready-to-use files you can copy into any GitHub Pages repository:

  • ✅ starter-template-index.html — A complete, beginner-friendly HTML homepage
  • ✅ starter-config.yml — A comprehensive Jekyll configuration file with detailed comments

🗂️ File 1: starter-template-index.html

A complete, beginner-friendly HTML template for your first GitHub Pages site. Drop it in, rename it, and you're live.

Features

  • ✅ Clean, modern design with gradient background
  • ✅ Fully responsive (mobile-friendly)
  • ✅ SEO-ready with meta tags
  • ✅ Social media preview tags (Open Graph)
  • ✅ Embedded CSS styling — no external frameworks required
  • ✅ Simple JavaScript example
  • ✅ Ready to customize in minutes

How to Use

  1. Copy starter-template-index.html to your repository root
  2. Rename it to index.html
  3. Replace the placeholder values:
    • [Your Name] → your actual name
    • [your interests] → your passions or hobbies
    • yourusername → your GitHub username
    • Project descriptions, skills, links, etc.
  4. Commit and push to GitHub
  5. Your site will be live at https://yourusername.github.io

⚙️ File 2: starter-config.yml

A comprehensive Jekyll configuration file with detailed comments explaining every option — no guesswork required.

Features

  • ✅ All essential Jekyll settings pre-configured
  • ✅ Theme configuration block
  • ✅ Plugin setup: SEO, sitemap, feed
  • ✅ Social media integration fields
  • ✅ SEO optimization settings
  • ✅ Custom variable examples
  • ✅ Extensive comments explaining each setting

How to Use

  1. Copy starter-config.yml to your repository root
  2. Rename it to _config.yml
  3. Customize the settings:
    • Update title, description, author, email
    • Set your url and baseurl
    • Add your social media usernames
    • Choose a theme and configure plugins as needed
  4. Commit and push to GitHub
  5. GitHub Pages will automatically use these settings

🚀 Quick Start Guide

Option 1: HTML-Only Site (No Jekyll)

Best for: Complete beginners and simple one-page sites.

  1. Copy starter-template-index.html → rename to index.html
  2. Customize the content
  3. Push to GitHub
  4. Done — your site is live!

Pros:

  • ✅ Simple and straightforward
  • ✅ No build process
  • ✅ Instant updates

Cons:

  • ❌ No templating (harder to maintain multiple pages)
  • ❌ No blog features
  • ❌ No Markdown support

Option 2: Jekyll-Powered Site

Best for: Blogs, multi-page sites, and content-focused projects.

  1. Rename starter-config.yml → _config.yml
  2. Create index.md or use a theme's default layout
  3. Choose a theme inside _config.yml
  4. Create posts in _posts/ (format: YYYY-MM-DD-title.md)
  5. Push to GitHub — Pages builds automatically

Pros:

  • ✅ Blog-awareness with automatic post listings
  • ✅ Markdown support (easier to write)
  • ✅ Templating with reusable layouts and includes
  • ✅ Themes for an instant professional design

Cons:

  • ❌ Slightly steeper learning curve
  • ❌ Longer build times
  • ❌ Configuration required

🎨 Customization Tips

Change Colors (starter-template-index.html)

Find this line in the <style> section and swap the hex values:

background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);

Try these palettes:

  • 🌊 Ocean: #2E3192 → #1BFFFF
  • 🔥 Fire: #FF512F → #DD2476
  • 🌿 Nature: #56AB2F → #A8E063
  • 🌸 Sunset: #FA709A → #FEE140

Add a New Section

Insert this snippet before the <footer> tag:

<section>
    <h2>New Section Title</h2>
    <p>Your content here...</p>
</section>

Add an Image

<img src="your-image.jpg" alt="Description" style="max-width: 100%; border-radius: 10px;">

Change Theme (starter-config.yml)

Uncomment and modify the theme line:

theme: jekyll-theme-cayman

Or use a remote theme:

remote_theme: pages-themes/architect@v0.2.0

Enable Google Analytics

Replace the placeholder with your tracking ID:

google_analytics: UA-XXXXXXXXX-X

Add a Custom Variable

custom_greeting: "Welcome to my site!"

Access it in any layout with: {% raw %}{{ site.custom_greeting }}{% endraw %}

🆘 Common Issues

My changes aren't showing up

  • ⏰ Wait 5–10 minutes for GitHub Pages to rebuild
  • 🔄 Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
  • 🔍 Check the Actions tab in your repository for build errors

Jekyll build failed

  • Check _config.yml for syntax errors — YAML is whitespace-sensitive!
  • Use 2 spaces for indentation, never tabs
  • Validate YAML at YAML Lint

404 Error

  • Repository name must match yourusername.github.io exactly (case-sensitive)
  • Confirm index.html or index.md exists in the repository root
