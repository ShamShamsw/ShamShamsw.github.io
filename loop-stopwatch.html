Loop Stopwatch

A self-resetting repeating timer with audio alerts — built entirely with vanilla JavaScript & Web Audio API.

Description

The Loop Stopwatch is a lightweight, dependency-free browser timer that counts up to a configurable duration, fires a mid-point audio alert, then automatically resets and restarts — incrementing a loop counter each cycle. It demonstrates the Web Audio API for programmatic sound generation, requestAnimationFrame for smooth 60 fps rendering, SVG progress arcs, and a clean JSON-driven configuration system. No server, no build step, no audio files — everything runs in a single HTML file.

Objectives Completed

  • Implemented a smooth, frame-accurate timer using performance.now() and requestAnimationFrame.
  • Added programmatic sound alerts via the Web Audio API — no external audio files required.
  • Built a mid-point chime (1:00) and a loop-reset triple-beep (1:30) with distinct tones.
  • Animated SVG progress ring that transitions color from indigo → pink as the deadline approaches.
  • Mute toggle so users can silence alerts without stopping the timer.
  • Fully documented customization guide: change durations, add checkpoints, retheme, and extend with new features.
  • JSON configuration schema covering all tuneable parameters.

How It Works

Code Architecture

The stopwatch is built from four cooperating pieces, all living inside one script block. Understanding each piece makes it easy to extend or repurpose the timer.

1 — Frame-accurate timing

Instead of drifting setInterval, the timer anchors every frame to a wall-clock snapshot captured when the user presses Start. Each animation frame recomputes elapsed time as performance.now() − startTime. This means pausing and resuming never accumulates error, and re-tabbing won't cause the display to jump. 

function tick() {
  if (!running) return;
  const now = performance.now();
  elapsed = now - startTime;          // always accurate to the wall clock

  if (elapsed >= TOTAL_MS) {          // loop boundary reached
    loopCount++;
    startTime = performance.now();    // reset anchor for next loop
    elapsed   = 0;
    firedOneMin = false;
  }

  updateDisplay(elapsed);
  rafId = requestAnimationFrame(tick); // schedule next frame (~16 ms)
}

2 — Alert checkpoints

Boolean flags (firedOneMin) guard each checkpoint so the callback fires exactly once per loop. The flag resets alongside elapsed when the loop restarts. 

// Fires once when the timer crosses the 1-minute mark
if (elapsed >= ONE_MIN_MS && !firedOneMin) {
  firedOneMin = true;
  fireFlash('#34d399', 'flash-one');
  playOneMinAlert();                   // ← your custom sound here
}

3 — SVG progress ring

The ring uses a single SVG <circle> whose stroke-dashoffset shrinks from the full circumference (753.98 px for r = 120) down to 0 as the loop progresses. CSS transition: stroke-dashoffset 0.1s linear keeps it visually smooth. 

const CIRCUMFERENCE = 2 * Math.PI * 120; // 753.98 px

function updateDisplay(ms) {
  const progress = Math.min(ms / TOTAL_MS, 1);
  arc.style.strokeDashoffset = CIRCUMFERENCE * (1 - progress);
  arc.style.stroke = ms >= ONE_MIN_MS ? '#f472b6' : '#818cf8';
}

4 — Web Audio API sounds

All audio is synthesized on-the-fly — no files to host. A shared AudioContext is created lazily on the first user interaction (browsers block audio until a gesture occurs). Each tone is an OscillatorNode routed through a GainNode for fade-out. 

function playTone(freq, startOffset, duration, type = 'sine', vol = 0.3) {
  const ctx = getAudioCtx();
  const osc  = ctx.createOscillator();
  const gain = ctx.createGain();
  osc.connect(gain);
  gain.connect(ctx.destination);
  osc.type            = type;
  osc.frequency.value = freq;
  gain.gain.setValueAtTime(vol, ctx.currentTime + startOffset);
  gain.gain.exponentialRampToValueAtTime(0.001,
      ctx.currentTime + startOffset + duration);
  osc.start(ctx.currentTime + startOffset);
  osc.stop(ctx.currentTime  + startOffset + duration + 0.05);
}

// 1-minute chime: C5 → E5 ascending
function playOneMinAlert() {
  playTone(523.25, 0,   0.28, 'sine', 0.25);
  playTone(659.25, 0.2, 0.35, 'sine', 0.2);
}

// Loop-reset beep: A4 · A4 · A5 triple
function playResetAlert() {
  playTone(440, 0,    0.1, 'square', 0.2);
  playTone(440, 0.18, 0.1, 'square', 0.2);
  playTone(880, 0.36, 0.22, 'sine',   0.3);
}
Customization

How to Make Changes

Tip: All tuneable numbers live near the top of the <script> block under a Configuration comment. You should never need to touch the rendering or sound engine to change timing or colours.

Change the loop duration

TOTAL_MS controls how long each loop runs in milliseconds. ONE_MIN_MS is the mid-point alert trigger. Both are plain numbers — easy to swap for any interval. 

// ── Configuration ────────────────────────────────────────
// Change these two numbers to match your use-case
const TOTAL_MS   = 90000;  // 1 min 30 sec  → 90 000 ms
const ONE_MIN_MS = 60000;  // alert at 1 min → 60 000 ms

// ── Examples ─────────────────────────────────────────────
// 25-minute Pomodoro with 20-minute warning
const TOTAL_MS   = 1500000; // 25 min
const ONE_MIN_MS =  1200000; // warn at 20 min

// 3-minute round, warn at 2 minutes (boxing / HIIT)
const TOTAL_MS   = 180000;
const ONE_MIN_MS = 120000;

Add extra checkpoints

Every checkpoint is just an if block in tick() guarded by a boolean flag. Add as many as you like — they all reset with the loop. 

// Declare flags alongside the other state variables
let firedHalfway = false;

// Inside tick() — add before the loop-reset block
if (elapsed >= TOTAL_MS / 2 && !firedHalfway) {
  firedHalfway = true;
  fireFlash('#f59e0b', 'flash-one');
  playTone(784, 0, 0.2, 'sine', 0.2);  // single G5 note
}

// Reset it alongside the other flags
firedHalfway = false;

Customise the sounds

playTone(freq, startOffset, duration, oscillatorType, volume) accepts any oscillator type (sine, square, sawtooth, triangle). Pitch is in Hz — use a piano frequency chart to pick notes. 

// Gentle descending lullaby chord (E5 → C5 → G4)
function playOneMinAlert() {
  if (!soundEnabled) return;
  playTone(659.25, 0,   0.3, 'sine', 0.22);
  playTone(523.25, 0.25,0.3, 'sine', 0.18);
  playTone(392,    0.5, 0.4, 'sine', 0.14);
}

// Aggressive buzzer (sawtooth low tones)
function playResetAlert() {
  if (!soundEnabled) return;
  playTone(110, 0,   0.15, 'sawtooth', 0.35);
  playTone(110, 0.2, 0.15, 'sawtooth', 0.35);
  playTone(110, 0.4, 0.3,  'sawtooth', 0.5);
}

Change the colour theme

Colours are hex strings in two places: the inline style attributes of the initial HTML, and the applyConfig() function. If you use the JSON config (below), update the config object and call applyConfig(config) on page load — the function handles the rest. 

const defaultConfig = {
  background_color:      '#0f1117',  // page background
  surface_color:         '#252837',  // ring border, button bg
  text_color:            '#e2e0ec',  // main timer digits
  primary_action_color:  '#818cf8',  // start button + arc colour
  secondary_action_color:'#6e6a7a',  // labels, muted text
};

Track lap history

Extend the loop callback to push a timestamp to an array and render a lap table. This turns the tool into a lap-timer for athletics, reaction-time practice, or process benchmarking. 

const laps = [];

// Inside the TOTAL_MS block in tick():
const lapTime = performance.now() - loopStartWall;
laps.push({ loop: loopCount, ms: lapTime });
renderLapTable(laps);           // append a <tr> to a nearby <table>
JSON Config

Configuration Schema

The configuration lives in json/loop-stopwatch.json. Load it with a fetch() call at startup and pass it to applyConfig(). This lets a CMS, a settings panel, or a URL parameter swap the entire stopwatch personality without touching any JavaScript.

Key Type Default Description
title_text string "Loop Stopwatch" Heading shown above the timer ring.
loop_duration_ms number 90000 Full loop length in milliseconds.
alert_one_ms number 60000 When the mid-point alert fires (ms into loop).
alert_one_color string "#34d399" Flash colour for the mid-point alert.
loop_reset_color string "#f472b6" Flash colour when the loop resets.
background_color string "#0f1117" Widget background.
surface_color string "#252837" Ring border, button backgrounds.
text_color string "#e2e0ec" Timer digit colour.
primary_action_color string "#818cf8" Start button and arc stroke colour.
secondary_action_colorstring "#6e6a7a" Labels and muted text.
font_family string "Outfit" UI font; any Google Font name works.
font_size number 16 Base font size in px (other sizes scale from this).
sound_enabled boolean true Master switch for all audio.
alert_one_sound string "chime" Sound preset for mid-point: "chime" or "beep".
loop_reset_sound string "triple_beep" Sound preset for loop reset: "triple_beep" or "buzz".
 
// Loading the config from JSON at page-start
fetch('json/loop-stopwatch.json')
  .then(r => r.json())
  .then(cfg => {
    // Merge with defaults so missing keys never cause errors
    const config = Object.assign({}, defaultConfig, cfg);
    applyConfig(config);
    // Override timing constants if the JSON specifies them
    if (config.loop_duration_ms) TOTAL_MS   = config.loop_duration_ms;
    if (config.alert_one_ms)     ONE_MIN_MS = config.alert_one_ms;
  });

Why It's Useful

Real-World Use Cases

The stopwatch's power comes from its simplicity. Because every behaviour is controlled by a handful of constants and two callback functions, you can adapt it to a surprising range of domains — each needing only a few lines changed.

🤷 Pomodoro / Study Blocks

Set TOTAL_MS = 1500000 (25 min) and ONE_MIN_MS = 1200000 (20 min warning). The loop counter tracks completed sessions automatically.

🏋 HIIT / Boxing Rounds

3-minute rounds with a 10-second warning: TOTAL_MS = 180000, ONE_MIN_MS = 170000. Distinct buzzer tone on reset signals rest time.

🎮 Game Show / Quiz Timer

60-second question rounds. The mid-point chime cues players that time is half up; the reset beep ends the round. No server required — runs from a static page.

🛠 Manufacturing / Process Check

Quality-check intervals on a factory floor. The visual flash is visible across a room; the audio alert works even without looking at the screen.

🌟 Meditation / Breathing

4-second inhale, 4-second hold, 8-second exhale — set three checkpoints with soft sine tones. The loop counter tracks completed breath cycles.

📋 Presentation Pacing

Per-slide time limits. The mid-point chime is a private signal to the speaker; the reset beep means move on. Embed it in a presenter overlay page.

Key flexibility point: because the timer drives behaviour through constants and callbacks rather than hard-coded logic, swapping from a Pomodoro to a boxing timer to a breathing guide is a 2-line config change — not a rewrite. The JSON schema means even non-developers can configure it through a settings UI backed by a simple form that writes the JSON file.
